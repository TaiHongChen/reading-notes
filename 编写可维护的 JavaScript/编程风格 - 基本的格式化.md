# 编程风格 - 基本的格式化

## 1.1 缩进层级

Zakas 推荐 4 个空格字符为一个缩进层级。（楼主个人喜好 2 个空格）

## 1.2 语句结尾 

Zakas 和 Crockford 都推荐在语句结尾不要省略封号的做法，不过现在不加封号的也越来越多，比如 Vue.js 以及 Zepto.js，个人还是习惯加封号，能避免一些低级错误，比如下面一个很经典的例子：

```javascript
return
{
  name: "hanzichi",
  age: 30
}
```

## 1.3 行的长度

Zakas 和 Crockford 都倾向于将行长度限定在 80 个字符。
